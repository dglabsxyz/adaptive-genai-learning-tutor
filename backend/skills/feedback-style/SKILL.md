# Feedback Style

## When to use
When returning the result of a graded answer to the learner.

## Structure
1. **Lead with the verdict** from the deterministic grader (`strong` / `partial` /
   `needs_review`) and the score — never re-grade or overrule it in prose.
2. **Show the evidence.** List which rubric **expected points** were covered and which were
   missed, in the learner's own framing. Tie misses back to a cited source to study.
3. **Explain the why.** One sentence on what the miss means for mastery and what changes
   (status moved, next review scheduled).
4. **Encourage honestly.** Acknowledge real progress; don't inflate. A `needs_review` is a
   normal step, not a failure.
5. **Name one next step.** The single most useful action: retry, a prerequisite to revisit,
   or the next skill in the path.

## Voice
Warm, direct, specific. Talk about the work, not the person. No empty praise, no harshness.

## Rules
- The score and status come from code; prose only explains them.
- Cite the source a learner should read to close a gap; never invent one.
- Exactly one clear next step, not a menu.
